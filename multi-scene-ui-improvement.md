# 多场景UI改进方案

## 1. 方案概述

本方案旨在改进多场景模式下的UI界面，提供更直观、易用的用户体验。采用统一的多场景架构，将过去的无场景模式映射为单场景的多场景模式，多场景模式永远激活，不再需要模式切换。

## 2. 核心设计

### 2.1 界面选择

**选择方案**：卡片式界面

**理由**：
- 适合场景数量较多的情况
- 可以快速浏览所有场景
- 空间利用更充分
- 在移动设备上也有较好的显示效果

### 2.2 卡片结构

每个场景卡片包含以下元素：

| 元素 | 描述 |
|------|------|
| 场景信息 | 场景序号、预计时长、实际音频时长 |
| 视觉需求 | 详细描述输入框 + 关键词标签（如 #城市 #夜景 #蓝色调） |
| 文案编辑 | 多行文本框，支持基本格式化 |
| 素材预览 | 显示已选择的图片/视频缩略图 |
| 操作按钮 | 预览、编辑、删除、复制、上移/下移 |

### 2.3 交互功能

- **场景管理**：添加、编辑、删除、复制场景
- **顺序调整**：通过上移/下移按钮调整场景顺序
- **批量操作**：全选、批量删除、批量复制
- **保存机制**：自动保存编辑内容，防止意外丢失
- **智能导入**：支持导入完整文案并自动分割为多场景
- **取消导入**：支持取消已导入的文案，不影响已解析的场景

## 3. 文案设置UI优化方案

### 3.1 问题分析

**原方案问题**：
- "基于视频主题"和"基于原始文案"两种方式混在一起，用户容易混淆
- 导入功能增加了UI复杂度
- 关键词生成按钮位置不够合理

**优化目标**：
- 将两种文案生成方式明确分离，互斥选择
- 简化UI，移除导入功能
- 优化关键词生成按钮位置

## 11. 用户已有完整文案处理方案

### 11.1 核心策略

**智能检测 + 条件处理 + 质量评价 + 自动化决策**

### 11.2 导入流程

1. **用户导入完整文案**：通过智能导入功能上传现有文案
2. **智能检测**：检测文案是否已按场景划分
3. **场景分割**：已划分的文案直接提取场景，未划分的自动分割
4. **质量评价**：评估分割结果的质量
5. **自动应用**：高质量结果自动应用，低质量结果提示用户调整
6. **场景管理**：用户可以在卡片式界面中进一步编辑和调整场景

### 11.3 处理流程

**步骤1：智能检测**
- 检测文案是否已按场景划分
- 检测场景标记、段落分割、时间标记

**步骤2：条件处理**
- **已划分文案**：直接提取场景信息
- **未划分文案**：调用LLM智能解析

**步骤3：质量评价**
- 场景数量合理性（3-8个场景最优）
- 时长分布合理性（3-15秒之间）
- 视觉需求完整性
- 关键词质量（3-5个关键词）
- 文案质量（20-200字之间）

**步骤4：三级决策**
- **自动接受（≥85分）**：直接创建场景卡片
- **自动拒绝（<60分）**：自动重试，最多3次
- **用户介入（60-85分）**：显示评价，用户选择

### 11.4 评分算法

```python
def evaluate_scenes(scenes):
    scores = {
        "scene_count": 0,
        "duration_distribution": 0,
        "visual_completeness": 0,
        "keyword_quality": 0,
        "script_quality": 0
    }
    
    # 场景数量评分
    scene_count = len(scenes)
    if 3 <= scene_count <= 8:
        scores["scene_count"] = 100
    elif 2 <= scene_count <= 10:
        scores["scene_count"] = 80
    elif 1 <= scene_count <= 15:
        scores["scene_count"] = 60
    else:
        scores["scene_count"] = 40
    
    # 时长分布评分
    durations = [s.get("duration", 0) for s in scenes]
    valid_durations = [d for d in durations if 3 <= d <= 15]
    duration_score = (len(valid_durations) / len(durations)) * 100
    scores["duration_distribution"] = duration_score
    
    # 视觉需求完整性评分
    visual_count = sum(1 for s in scenes if s.get("visual_requirement"))
    visual_score = (visual_count / len(scenes)) * 100
    scores["visual_completeness"] = visual_score
    
    # 关键词质量评分
    keyword_count = sum(1 for s in scenes if s.get("keywords"))
    keyword_score = (keyword_count / len(scenes)) * 100
    scores["keyword_quality"] = keyword_score
    
    # 文案质量评分
    script_count = sum(1 for s in scenes if s.get("script") and 20 <= len(s["script"]) <= 200)
    script_score = (script_count / len(scenes)) * 100
    scores["script_quality"] = script_score
    
    # 综合评分
    total_score = sum(scores.values()) / len(scores)
    
    return {
        "total_score": total_score,
        "individual_scores": scores,
        "auto_accept": total_score >= 85,  # 85分以上自动接收
        "auto_reject": total_score < 60   # 60分以下自动拒绝
    }
```

### 11.5 重试机制

```python
def auto_parse_with_retry(script, max_retries=3):
    for attempt in range(max_retries):
        # 调用LLM解析
        scenes = llm.parse_script(script)
        
        # 评价识别结果
        evaluation = evaluate_scenes(scenes)
        
        # 决策
        if evaluation["total_score"] >= 85:
            # 自动接受
            return {"status": "success", "scenes": scenes}
        elif evaluation["total_score"] < 60:
            # 自动拒绝，重试
            logger.warning(f"Parse attempt {attempt+1}: score {evaluation['total_score']}, retrying...")
            continue
        else:
            # 需要用户介入
            return {"status": "manual", "scenes": scenes, "evaluation": evaluation}
    
    # 所有重试都失败
    return {"status": "failed", "message": "无法达到识别质量标准，请手动创建场景"}
```
