# 视频合并顺序规范

## 1. 概述

本文档详细说明 MoneyPrinterTurbo 中视频合并的顺序逻辑，包括场景级别和目标视频级别的视频段合并规则。

## 2. 视频合并层次结构

### 2.1 核心概念

- **场景（Scene）**：视频的基本组成单位，每个场景包含一个或多个视频片段
- **视频片段（Video Clip）**：单个视频文件或其部分内容
- **片头视频（Intro Video）**：视频开头的特殊视频片段
- **普通视频段（Regular Video Clips）**：场景中的常规视频片段

### 2.2 合并层次

1. **场景内部合并**：处理单个场景内的视频片段
2. **场景间合并**：按场景顺序合并所有场景
3. **最终合成**：添加音频和字幕（目标视频级别）

## 3. 视频连接模式（VideoConcatMode）

### 3.1 定义

视频连接模式控制**场景内部**视频片段的连接方式，不影响场景间的顺序。

### 3.2 可用模式

- **顺序模式（sequential）**：按照视频片段的原始顺序连接
- **随机模式（random）**：随机打乱场景内的视频片段顺序

### 3.3 作用范围

- **全局配置**：视频连接模式是全局统一配置，适用于所有场景
- **仅影响**：场景内部的普通视频片段
- **不影响**：
  - 场景间的顺序
  - 最终合成顺序
  - 字幕合成

## 4. 合并顺序规则

### 4.1 最终合成顺序

1. **场景视频**：按照场景顺序依次排列
2. **场景内视频**：根据视频连接模式处理

### 4.2 场景级别处理

对于每个场景：

1. 提取场景中的所有视频片段
2. 根据视频连接模式处理片段顺序：
   - 顺序模式：保持原始顺序
   - 随机模式：打乱片段顺序
3. 处理视频片段（调整大小、应用效果等）
4. 合并为场景级视频

### 4.3 目标视频级别处理

1. 按场景顺序处理每个场景的视频
2. 合并所有场景视频
3. 添加音频（外部音频或场景视频的原始音频）
4. 添加字幕（如果有）
5. 生成最终视频

### 4.4 音频处理原则

- **默认音频**：直接使用场景视频中的原始音频
- **背景音频**：如果添加背景音频，基本原则是音频适配视频时长
- **同步处理**：确保音频与视频保持同步，避免音频混乱

### 4.5 字幕处理原则

- **字幕来源**：可以是外部字幕文件或自动生成的字幕
- **字幕添加**：在最终合成阶段添加字幕
- **同步处理**：确保字幕与视频内容保持同步
- **样式控制**：支持自定义字幕样式、位置和大小

## 5. 实现差距分析

### 5.1 当前实现问题

1. **场景概念缺失**：当前实现没有明确的场景级别概念，所有普通视频片段被视为一个整体
2. **顺序控制不当**：随机模式会打乱所有视频片段，破坏场景顺序
3. **连接模式作用范围**：连接模式影响了整个视频集，而不仅仅是场景内部

### 5.2 期望实现

1. **场景级别处理**：按场景分组处理视频片段
2. **保持场景顺序**：场景间严格按照原始顺序排列
3. **连接模式限制**：连接模式仅影响场景内部的视频片段顺序
4. **音频处理优化**：确保音频与视频同步，避免音频混乱

### 5.3 实现逻辑细节

根据当前实现，视频合成的核心逻辑如下：

1. **音频时长计算**：首先计算音频时长或视频总时长
2. **视频处理**：处理每个视频文件，分割成子剪辑并应用效果
3. **视频合并**：按顺序拼接处理后的视频剪辑
4. **音频合并**：添加音频（外部音频或场景视频的原始音频）
5. **最终输出**：生成最终视频文件

当前实现遵循音频适配视频的原则，确保音频与视频保持同步。

## 6. 技术实现建议

### 6.1 数据结构调整

- 引入场景级别的数据结构，将视频片段按场景分组
- 为每个场景存储其视频片段和连接模式

### 6.2 合并逻辑优化

1. **场景预处理**：
   - 按场景分组视频片段
   - 对每个场景应用连接模式

2. **场景间合并**：
   - 按场景顺序合并处理后的场景视频
   - 保持场景顺序不变

3. **最终处理**：
   - 添加音频（外部音频或场景视频的原始音频）
   - 添加字幕（如果有）
   - 确保音频与视频同步
   - 确保字幕与视频内容同步

### 6.3 代码修改建议

- **函数分离**：使用不同的函数处理不同级别的视频合并，便于维护和扩展
  - `process_scene_videos`：处理场景级别的视频合并
  - `combine_scenes`：处理场景间的合并
  - `finalize_video`：处理最终合成（添加音频、字幕等）
- **场景级别处理**：按场景分组处理视频片段
- **连接模式应用**：确保全局统一的连接模式仅应用于场景内部
- **字幕集成**：在最终合成阶段集成字幕处理

## 7. 示例流程

### 7.1 输入

- **全局视频连接模式**：随机（适用于所有场景）
- 场景 1：视频片段 A、B、C
- 场景 2：视频片段 D、E、F
- 音频：`audio.mp3`
- 字幕：`subtitle.srt`（可选）

### 7.2 处理流程

1. 处理场景 1：
   - 应用全局连接模式（随机）：打乱视频片段为 B、A、C
   - 处理并合并为场景 1 视频
2. 处理场景 2：
   - 应用全局连接模式（随机）：打乱视频片段为 F、D、E
   - 处理并合并为场景 2 视频
3. 按顺序合并：场景 1 + 场景 2
4. 添加音频（适配视频时长）
5. 添加字幕（如果有）
6. 生成最终视频

### 7.3 输出

最终视频顺序：B → A → C → F → D → E

## 8. 总结

视频合并顺序应遵循以下原则：

1. **保持场景顺序**：场景间严格按照原始顺序排列
2. **灵活处理场景内部**：通过连接模式控制场景内视频片段的顺序
3. **统一最终合成**：按场景顺序的方式合成最终视频
4. **音频适配视频**：音频处理应适配视频时长，确保同步

通过正确实现这些规则，可以确保视频合成结果符合预期，同时保持场景的逻辑连贯性。

## 9. Roadmap

### 9.1 未来功能规划

- **目标视频级别的片头**：在目标视频级别添加片头功能，允许用户为整个视频添加统一的片头
- **场景级别的音频控制**：为每个场景添加独立的音频处理选项
- **智能音频同步**：优化音频与视频的同步处理，避免音频混乱
- **高级转场效果**：为场景间添加更丰富的转场效果，同时确保音频平滑过渡
