# 多场景模式实现方案

## 1. 功能概述

多场景模式是一种视频生成增强功能，它将视频内容划分为多个独立场景，每个场景有自己的视觉需求和主播文案，最终将所有场景片段组合成完整视频。

## 3. 后端实现方案

### 3.1 数据结构设计

#### 场景数据结构
```python
class Scene:
    def __init__(self):
        self.id = str  # 场景ID
        self.start_time = float  # 开始时间(秒)
        self.end_time = float  # 结束时间(秒)
        self.camera = str  # 镜头描述
        self.title = str  # 场景标题
        self.script = str  # 主播文案
        self.keywords = list  # 视频关键词列表
        self.video_clips = list  # 视频片段列表
        self.audio_file = str  # 音频文件路径
        self.subtitle_file = str  # 字幕文件路径
```

#### 多场景任务数据结构

class MultiSceneTask:

### 3.2 处理流程

#### 1. 文案处理流程

**当用户提供主题时：**
1. 检测多场景模式是否激活
2. 如果激活，调用LLM生成多场景文案
3. 解析多场景文案，提取场景信息
4. 为每个场景生成视频关键词

**当用户提供文案时：**
1. 检测多场景模式是否激活
2. 如果激活，调用LLM将文案转换为多场景格式
3. 解析多场景文案，提取场景信息
4. 为每个场景生成视频关键词

#### 2. 视频生成流程

1. 遍历每个场景
2. 对每个场景：
   - 根据场景关键词搜索视频素材
   - 生成场景音频
   - 生成场景字幕
   - 合成场景视频片段（不含背景音乐）
3. 收集所有场景片段
4. 调用现有视频服务的最终合成功能
5. 添加背景音乐，生成最终视频

### 3.3 核心算法

#### 多场景文案解析算法
1. 识别场景分隔符（如`(镜头:`标记）
2. 识别时间标记（如`【0-5秒`）
3. 提取场景标题和内容
4. 构建场景对象列表

#### 场景关键词生成算法
1. 分析场景的镜头描述和文案内容
2. 提取关键概念和实体
3. 生成适合视频搜索的关键词列表

#### 场景视频合成算法
1. 为每个场景单独执行现有视频生成流程的前几步
2. 跳过背景音乐添加步骤
3. 收集所有场景片段
4. 按顺序组合片段并添加背景音乐

## 6. 注意事项

1. **兼容性**：
   - 确保多场景模式关闭时，系统行为与现有版本完全一致
   - 保持API向后兼容

2. **性能考虑**：
   - 多场景模式会增加处理时间和资源消耗
   - 考虑添加并发处理机制
   - 实现场景级的错误处理和重试机制

3. **用户体验**：
   - 提供清晰的多场景文案格式说明
   - 显示详细的场景处理进度
   - 支持手动调整场景分割和关键词

4. **边缘情况**：
   - 处理文案格式不正确的情况
   - 处理场景数量过多的情况
   - 处理场景视频素材不足的情况
