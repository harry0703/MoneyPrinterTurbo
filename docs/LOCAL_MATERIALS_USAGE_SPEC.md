# 本地素材语义匹配系统

## 1. 功能概述

**适用范围：本文档仅适用于用户选择素材源为本地的情况。**

本地素材语义匹配系统是 Coiner 的核心功能之一，旨在通过智能匹配算法，将本地视频素材与场景语义进行关联，提升视频生成的相关性和质量。系统遵循本地素材优先原则，确保用户指定的本地素材能够优先使用，并且每个场景都从本地素材开始播放。

## 2. 核心原则

### 2.1 本地素材优先
- **优先使用本地素材**：当用户指定本地素材时，系统优先使用本地素材
- **线上素材作为补充**：仅当本地素材不足时，才使用线上素材作为补充
- **场景开头必须使用本地素材**：每个场景的视频必须从本地素材开始播放

### 2.2 语义匹配
- **基于关键词匹配**：本地素材与场景关键词进行语义匹配
- **匹配度排序**：按匹配度分数降序排序本地素材
- **智能匹配算法**：
  - 完全匹配（关键词在文件名中）：10分
  - 部分匹配（关键词与文件名相互包含）：5分

### 2.3 错误处理
- **明确报错**：如果需要本地素材但匹配不到，系统会明确报错
- **详细错误信息**：提供清晰的错误原因和解决建议
- **任务状态更新**：当本地素材缺失时，任务状态标记为失败

## 3. 实现方案

### 3.1 图片转视频功能

**目标：** 将本地图片转换为3-5秒的视频片段

**实现位置：** `app/services/video.py` - `preprocess_video` 函数

**技术实现：**
- 支持固定时长和随机时长两种模式
- 随机时长模式下，为每个图片生成3-5秒的随机时长
- 使用 MoviePy 的 `ImageClip` 将图片转换为视频
- 自动创建临时视频文件并更新素材信息

**调用方式：**
```python
materials = video.preprocess_video(
    materials=params.video_materials, clip_duration=(3, 5)
)
```

**日志输出：**
```
Converted image to video: landscape.jpg -> landscape.jpg.mp4 (4.2s)
```

### 3.2 语义匹配算法

**目标：** 基于场景关键词匹配本地视频素材

**实现位置：** `app/services/video.py` - `match_local_videos_by_keywords` 函数

**匹配规则：**
1. **完全匹配**（关键词在文件名中）：10分
2. **部分匹配**（关键词与文件名相互包含）：5分

**实现流程：**
1. 规范化场景关键词（转为小写并去空格）
2. 遍历本地素材，计算每个素材的匹配分数
3. 按匹配分数降序排序
4. 返回排序后的素材列表

**算法核心：**
```python
# 完全匹配（关键词在文件名中）
if keyword in filename:
    score += 10

# 部分匹配（关键词与文件名相互包含）
if keyword in filename or filename in keyword:
    score += 5
```

**匹配结果示例：**
```
Matched 5 materials by keywords: ['nature', 'forest', 'mountain']
  1. nature_forest.mp4 (score: 20)
  2. mountain_view.jpg.mp4 (score: 15)
  3. forest_scene.mp4 (score: 10)
  4. river_nature.mp4 (score: 10)
  5. landscape.jpg.mp4 (score: 5)
```

### 3.3 场景语义匹配优先

**目标：** 为每个场景优先使用语义匹配度最高的视频素材

**实现位置：** `app/services/task.py` - `process_scene` 函数

**工作流程：**
1. 预处理本地素材（图片转视频）
2. 基于场景关键词匹配和排序素材
3. 场景开始使用匹配度最高的视频
4. 后续使用匹配度次高的视频

**实现代码：**
```python
# 预处理本地素材
materials = video.preprocess_video(
    materials=params.video_materials, clip_duration=(3, 5)
)

# 匹配本地视频与场景关键词
if materials and scene_keywords:
    materials = video.match_local_videos_by_keywords(materials, scene_keywords)

# 使用匹配后的素材
downloaded_videos = [m.url for m in materials]
```

## 4. 工作流程

### 4.1 本地素材处理
1. **素材预处理**：
   - 图片自动转换为3-5秒的视频片段
   - 视频素材直接使用

2. **语义匹配**：
   - 提取场景关键词
   - 计算本地素材与关键词的匹配度
   - 按匹配度排序本地素材

3. **本地素材选择**：
   - 选择匹配度最高的本地素材作为场景开头
   - 其他本地素材按匹配度顺序使用

### 4.2 线上素材补充
1. **补充条件**：
   - 本地素材数量不足
   - 本地素材总时长不够

2. **补充策略**：
   - 下载与场景关键词相关的线上素材
   - 线上素材排在本地素材之后
   - 确保视频总时长满足需求

### 4.3 素材组合
- **本地素材在前**：确保每个场景的视频从本地素材开始
- **线上素材在后**：线上素材作为补充，排在本地素材之后
- **无缝衔接**：本地与线上素材之间平滑过渡

## 5. 技术实现

### 5.1 核心函数

#### `preprocess_video` 函数
- **功能**：预处理本地素材，将图片转换为视频
- **参数**：
  - `materials`：MaterialInfo对象列表
  - `clip_duration`：视频片段时长（支持固定值或3-5秒随机范围）
- **返回值**：处理后的MaterialInfo对象列表

#### `match_local_videos_by_keywords` 函数
- **功能**：根据场景关键词匹配本地素材
- **参数**：
  - `materials`：MaterialInfo对象列表
  - `scene_keywords`：场景关键词列表
- **返回值**：按匹配度排序的MaterialInfo对象列表

#### `process_scene` 函数
- **功能**：处理单个场景的视频素材
- **流程**：
  1. 预处理本地素材
  2. 语义匹配本地素材
  3. 下载线上素材作为补充
  4. 组合本地和线上素材
  5. 生成场景视频

#### `get_video_materials` 函数
- **功能**：获取视频素材（本地优先，线上补充）
- **流程**：
  1. 处理本地素材
  2. 语义匹配
  3. 下载线上补充素材
  4. 组合素材

### 5.2 配置参数

| 参数 | 描述 | 默认值 |
|------|------|--------|
| `video_source` | 视频源选择 | "pexels" |
| `video_materials` | 本地素材列表 | None |
| `video_clip_duration` | 视频片段时长 | 5 |

## 6. 技术优势

### 6.1 向后兼容
- 仍支持固定时长模式
- 不影响现有功能

### 6.2 性能优化
- 轻量级匹配算法，不影响处理速度
- 仅在本地素材模式下启用
- 并行处理：支持多线程处理视频素材
- 内存释放：定期释放内存，避免内存泄漏

### 6.3 可扩展性
- 易于添加更多匹配规则
- 支持未来集成更复杂的语义分析

### 6.4 用户友好
- 详细的日志输出，便于调试
- 自动处理图片转视频，无需用户干预

## 7. 错误处理

### 7.1 常见错误

| 错误类型 | 错误信息 | 解决方法 |
|----------|----------|----------|
| 本地素材缺失 | "No local materials found. When local materials are provided, they must be used for scene start." | 确保选择了有效的本地素材文件 |
| 本地素材格式错误 | "Failed to convert image to video: {material.url} -> {str(e)}" | 检查图片文件是否损坏或格式不支持 |
| 线上素材下载失败 | "failed to download videos, maybe the network is not available." | 检查网络连接或使用VPN |

### 7.2 错误恢复
- **本地素材缺失**：任务失败，需要重新选择本地素材
- **线上素材下载失败**：如果本地素材足够，任务继续；否则任务失败

## 9. 配置与使用

### 9.1 配置参数
- **clip_duration**：图片转视频的时长，支持固定值或范围元组
- **video_source**：设置为 "local" 启用本地素材模式

### 9.2 使用指南

1. **选择本地素材**：
   - 在Web界面选择本地视频/图片文件
   - 支持多个文件同时选择
   - 视频格式：MP4、AVI、MOV等常见格式
   - 图片格式：PNG、JPG、JPEG、BMP、GIF
   - 文件大小：建议单个文件不超过100MB

2. **设置关键词**：
   - 为每个场景添加相关的关键词
   - 具体描述场景内容的词语
   - 与本地素材文件名相关的词语
   - 多个关键词用逗号分隔

3. **开始生成**：
   - 系统会自动处理本地素材并进行语义匹配
   - 日志会显示本地素材匹配情况和线上素材补充情况
   - 生成的视频会从本地素材开始播放

## 13. 总结

本地素材语义匹配系统通过智能算法将本地素材与场景语义关联，显著提升了视频生成的质量和相关性。系统不仅实现了图片到视频的自动转换，还通过文件名匹配为每个场景选择最适合的视频素材，为用户提供了更加智能和个性化的视频生成体验。

本地素材优先原则确保了用户指定的本地素材能够优先使用，并且每个场景都从本地素材开始播放。当本地素材不足时，系统会自动下载线上素材作为补充，确保视频长度满足需求。

这种设计既尊重了用户的选择，又保证了视频生成的灵活性和可靠性，为用户提供了更加个性化和高质量的视频生成体验。
