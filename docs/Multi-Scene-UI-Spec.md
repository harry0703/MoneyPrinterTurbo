# 多场景模式实现方案

## 1. 功能概述

多场景模式是一种视频生成增强功能，它将视频内容划分为多个独立场景，每个场景有自己的视觉需求和主播文案，最终将所有场景片段组合成完整视频。采用统一的多场景架构，多场景模式永远激活，不再需要模式切换。

## 3. 后端实现方案

### 3.1 数据结构设计

#### 场景数据结构
```python
{
    "id": str,                    # 场景唯一ID
    "duration": int,              # 场景时长（秒）
    "visual_requirement": str,    # 视觉需求描述
    "keywords": str,              # 视频关键词（逗号分隔）
    "script": str                 # 场景文案
}
```

#### Session State 状态管理
```python
# 初始化状态
if "video_script" not in st.session_state:
    st.session_state["video_script"] = ""

if "video_subject" not in st.session_state:
    st.session_state["video_subject"] = ""

if "video_terms" not in st.session_state:
    st.session_state["video_terms"] = ""

if "scenes" not in st.session_state:
    st.session_state["scenes"] = []

if "auto_parse_mode" not in st.session_state:
    st.session_state["auto_parse_mode"] = "auto"  # "auto" or "manual"

if "pending_scenes" not in st.session_state:
    st.session_state["pending_scenes"] = []

if "pending_evaluation" not in st.session_state:
    st.session_state["pending_evaluation"] = {}
```

### 3.2 处理流程

#### 1. 基于视频主题流程

1. 输入视频主题
2. 选择脚本语言
3. 点击"根据主题生成视频文案和关键词"按钮
4. AI生成文案并保存到 session_state["video_script"]
5. 用户可以修改生成的文案
6. 用户可以点击"解析当前文案"按钮生成场景
7. 用户可以在场景管理区域编辑场景

#### 2. 基于原始文案流程

1. 直接在视频文案文本域输入或粘贴文案
2. 点击"解析当前文案"按钮
3. 解析文案并创建场景
4. 用户可以在场景管理区域编辑场景
5. 用户可以点击"根据文案生成视频关键词"按钮

#### 3. 视频生成流程

1. 遍历每个场景
2. 对每个场景：
   - 根据场景关键词搜索视频素材
   - 生成场景音频
   - 生成场景字幕
   - 合成场景视频片段
3. 收集所有场景片段
4. 调用现有视频服务的最终合成功能
5. 添加背景音乐，生成最终视频

### 3.3 核心算法

#### 场景解析算法

调用 `auto_parse_script` 函数解析文案：

```python
result = auto_parse_script(
    script,
    max_retries=3,
    auto_mode=True
)
```

返回结果：
- `status`: "success" | "manual" | "failed"
- `scenes`: 解析后的场景列表
- `evaluation`: 质量评价结果

#### 场景管理操作

**添加场景**：
```python
new_scene = {
    "id": str(uuid4()),
    "script": "",
    "visual_requirement": "",
    "keywords": "",
    "duration": 5
}
st.session_state["scenes"].append(new_scene)
```

**删除场景**：
```python
st.session_state["scenes"].pop(index)
```

**复制场景**：
```python
copied_scene = scene.copy()
copied_scene["id"] = str(uuid4())
st.session_state["scenes"].append(copied_scene)
```

**调整顺序**：
```python
# 上移
st.session_state["scenes"][i], st.session_state["scenes"][i-1] = \
    st.session_state["scenes"][i-1], st.session_state["scenes"][i]

# 下移
st.session_state["scenes"][i], st.session_state["scenes"][i+1] = \
    st.session_state["scenes"][i+1], st.session_state["scenes"][i]
```

## 6. 注意事项

1. **兼容性**：
   - 多场景模式始终启用
   - 向后兼容单场景数据

2. **性能考虑**：
   - 场景数量过多时考虑分页或虚拟滚动
   - 优化场景解析性能

3. **用户体验**：
   - 提供清晰的操作引导
   - 场景为空时显示占位提示
   - 支持场景的快速编辑

4. **边缘情况**：
   - 处理空文案情况
   - 处理解析失败情况
   - 处理场景数据异常

## 7. 预期效果

启用多场景模式后，视频生成质量将显著提升：
- 每个场景都有独立的视觉需求和文案
- 视频内容更加丰富多样
- 生成的视频更加专业和吸引人
- 用户可以灵活管理每个场景

## 10. 风险评估

1. **技术风险**：
   - LLM生成的文案质量可能不稳定
   - 场景解析可能存在误差

2. **用户体验风险**：
   - 场景管理可能增加操作复杂度

3. **解决方案**：
   - 提供场景编辑功能，允许用户手动调整
   - 优化解析算法，提高准确性
   - 提供清晰的操作引导

## 11. 结论

多场景模式是对现有视频生成系统的重要增强，它通过结构化的场景划分和针对性的视觉内容生成，能够显著提升生成视频的质量和专业性。采用统一的多场景架构，消除了模式切换的复杂性，为用户提供了一致的操作体验。

通过卡片式界面设计，用户可以直观地管理每个场景，灵活调整场景内容和顺序。这种设计不仅简化了代码结构，也为未来功能扩展预留了更大的空间。
